Dynamic Method Dispatch, also known as **runtime polymorphism**, is a mechanism in object-oriented programming where the method to be executed is determined at runtime rather than compile time. This feature is central to achieving polymorphism in Java and allows for flexibility and extensibility in code.

When a subclass overrides a method from its superclass, the overridden method is executed based on the **actual type of the object** being referred to, not the type of the reference variable. This is achieved through **upcasting**, where a superclass reference variable refers to a subclass object.

Syntax and Example

Here is an example to demonstrate dynamic method dispatch:
```java
class A {
   void m1() {
       System.out.println("Inside A's m1 method");
   }
}
class B extends A {
   void m1() {
       System.out.println("Inside B's m1 method");
   }
}
class C extends A {
   void m1() {
       System.out.println("Inside C's m1 method");
   }
}
public class Dispatch {
   public static void main(String[] args) {
       A a = new A(); // Object of type A
       B b = new B(); // Object of type B
       C c = new C(); // Object of type C
       A ref; // Reference of type A
       ref = a;
       ref.m1(); // Calls A's m1 method
       ref = b;
       ref.m1(); // Calls B's m1 method
       ref = c;
       ref.m1(); // Calls C's m1 method
   }
}

```
**Output**:

Inside A's m1 method
Inside B's m1 method
Inside C's m1 method
