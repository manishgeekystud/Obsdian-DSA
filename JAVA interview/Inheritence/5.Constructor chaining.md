# Constructor Chaining in Java (Interview Notes)

> **Interview Frequency:** ⭐⭐⭐⭐⭐ (Very Frequently Asked)

Constructor Chaining is one of the most important inheritance concepts and is frequently asked in Java interviews.

---

# What is Constructor Chaining?

## Definition

**Constructor Chaining** is the process of **calling one constructor from another constructor** during object creation.

The purpose is to **reuse constructor code** and **initialize objects in the correct order**.

---

# Types of Constructor Chaining

There are two types:

|Type|Keyword Used|
|---|---|
|Within the same class|`this()`|
|Between parent and child classes|`super()`|

---

# 1. Constructor Chaining using `this()`

Used to call **another constructor of the same class**.

## Example

```
class Student {

    Student() {
        this(101);
        System.out.println("Default Constructor");
    }

    Student(int id) {
        System.out.println("Parameterized Constructor");
    }

    public static void main(String[] args) {
        new Student();
    }
}
```

### Output

```
Parameterized Constructor
Default Constructor
```

---

## Constructor Call Flow

```
new Student()

        │

        ▼

Student()

        │

    this(101)

        │

        ▼

Student(int)

        │

        ▼

Back to Student()
```

---

# 2. Constructor Chaining using `super()`

Used to call the **parent class constructor**.

## Example

```
class Animal {

    Animal() {
        System.out.println("Animal Constructor");
    }
}

class Dog extends Animal {

    Dog() {
        super();
        System.out.println("Dog Constructor");
    }

    public static void main(String[] args) {
        new Dog();
    }
}
```

### Output

```
Animal Constructor
Dog Constructor
```

---

## Constructor Call Flow

```
new Dog()

        │

        ▼

Dog()

        │

     super()

        │

        ▼

Animal()

        │

        ▼

Back to Dog()
```

# Complete Constructor Chaining Example

```
class Animal {

    Animal() {
        System.out.println("Animal Constructor");
    }
}

class Dog extends Animal {

    Dog() {
        this(100);

    }

    Dog(int x) {
        super();
        System.out.println("Parameterized Dog Constructor");
    }

    public static void main(String[] args) {
        new Dog();
    }
}
```

# Rules of Constructor Chaining

### Rule 1

`this()` must be the **first statement** in the constructor.

✅ Correct

```
Student() {
    this(100);
}
```

❌ Wrong

```
Student() {
    System.out.println("Hi");
    this(100);      // Compilation Error
}
```

---

### Rule 2

`super()` must also be the **first statement**.

```
Dog() {
    super();
}
```

---

### Rule 3

A constructor can have **either**

```
this();
```

OR

```
super();
```

Never both.

Wrong

```
Dog() {

    this();

    super();     // Compilation Error
}
```

Reason:

Both must be the first statement.

---

### Rule 4

If you don't write

```
super();
```

Java inserts it automatically **only if** the parent has a no-argument constructor.

---

# Constructor Execution Order

Suppose

```
A
↑
B
↑
C
```

```
new C();
```

Execution order

```
A()

↓

B()

↓

C()
```

The parent constructor always executes before the child constructor.

---

# Why Constructor Chaining?

Without constructor chaining:

- Code duplication increases.
- Object initialization becomes inconsistent.
- Parent class may not be initialized properly.

Constructor chaining ensures every part of the object is initialized in the correct order.

---

# Difference: `this()` vs `super()`

|`this()`|`super()`|
|---|---|
|Calls another constructor in the same class|Calls the parent class constructor|
|Used for constructor reuse|Used for parent initialization|
|Same class|Parent class|
|Must be first statement|Must be first statement|
|Cannot be used with `super()` in the same constructor|Cannot be used with `this()` in the same constructor|

# Common Interview Questions

### 1. What is Constructor Chaining?

Constructor chaining is the process of calling one constructor from another constructor using `this()` or `super()`.

---

### 2. Why is Constructor Chaining used?

- Reuse constructor code.
- Avoid duplication.
- Ensure proper initialization.
- Improve maintainability.

---

### 3. Can we use both `this()` and `super()` in the same constructor?

❌ No.

Both must be the first statement.

---

### 4. What happens if we don't write `super()`?

The compiler inserts `super()` automatically **only if** the parent has a no-argument constructor.

---

### 5. Can constructors be inherited?

❌ No.

Constructors are **not inherited**, but child constructors can call parent constructors using `super()`.

---

### 6. In what order are constructors executed?

From the **topmost parent** down to the **child**.

Example:

```
Object()

↓

Parent()

↓

Child()
```

---

# Common Mistakes

### ❌ Mistake 1

Calling `this()` after another statement.

```
Student() {

    System.out.println("Hello");

    this(100);
}
```

Compilation Error.

---

### ❌ Mistake 2

Calling both `this()` and `super()`.

```
Student() {

    this();

    super();
}
```

Compilation Error.

---

### ❌ Mistake 3

Assuming `super()` is always optional.

If the parent class has **only parameterized constructors**, you **must** explicitly call the appropriate constructor with `super(arguments)`.
