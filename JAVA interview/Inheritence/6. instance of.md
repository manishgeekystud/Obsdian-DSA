In Java, `instanceof` is an operator used to check whether an object is an instance of a particular class, subclass, or interface.

### Syntax

```
object instanceof ClassName
```

- Returns `true` if `object` is an instance of `ClassName` (or its subclass/implements the interface).
- Returns `false` otherwise.

### Example 1: Basic Usage

```
class Animal {}

class Dog extends Animal {}

public class Main {
    public static void main(String[] args) {
        Animal animal = new Dog();

        System.out.println(animal instanceof Dog);     // true
        System.out.println(animal instanceof Animal);  // true
        System.out.println(animal instanceof Object);  // true
    }
}
```

### Example 2: Safe Type Casting

`instanceof` is commonly used before casting an object.

```
class Animal {
    void sound() {
        System.out.println("Animal sound");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Woof!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal animal = new Dog();

        if (animal instanceof Dog) {
            Dog dog = (Dog) animal;
            dog.bark();
        }
    }
}
```

### Example 3: With `null`

```
String str = null;

System.out.println(str instanceof String); // false
```

`instanceof` always returns `false` if the reference is `null`.

### Java 16+ Pattern Matching

Modern Java allows you to combine the type check and cast:

```
Animal animal = new Dog();

if (animal instanceof Dog dog) {
    dog.bark();   // No explicit cast needed
}
```

This is equivalent to:

```
if (animal instanceof Dog) {
    Dog dog = (Dog) animal;
    dog.bark();
}
```

### Summary

|Expression|Result|
|---|---|
|`obj instanceof ClassName`|`true` if `obj` is an instance of `ClassName` or its subclass|
|`null instanceof ClassName`|`false`|
|Common use|Safe type checking before casting|

`instanceof` helps prevent `ClassCastException` by ensuring an object is of the expected type before casting.
