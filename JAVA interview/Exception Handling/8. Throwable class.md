> **`Throwable` is the root class of Java's exception hierarchy. Every object that can be thrown using the `throw` statement must be an instance of `Throwable` or one of its subclasses.**

Simply put:

- `Throwable` is the **parent class** of all exceptions and errors.
- Every exception handling mechanism in Java is built around `Throwable`.

---

# Exception Hierarchy

```
                    Object
                       |
                  Throwable
                 /          \
              Error      Exception
                            |
                    RuntimeException
```

Remember this hierarchy. It is one of the most commonly asked interview diagrams.

---

# Why is Throwable Needed?

Suppose Java had only the `Exception` class.

Then JVM errors like

- OutOfMemoryError
- StackOverflowError

could not be represented separately.

So Java introduced:

```
Throwable
      |
---------------------
|                   |
Error          Exception
```

This separates:

- **Recoverable problems** → Exception
- **Serious JVM problems** → Error

# Components of Throwable

```
Throwable
   |
----------------------------
| Message                  |
| Cause                    |
| Stack Trace              |
| Suppressed Exceptions    |
----------------------------
```

Every exception object inherits these features.

# Frequently Asked Interview Questions

## 1. What is Throwable?

**Answer:**

`Throwable` is the root class of Java's exception hierarchy. Every object that can be thrown or caught must extend `Throwable`.

---

## 2. What are the subclasses of Throwable?

Two direct subclasses:

- `Exception`
- `Error`

---

## 3. Can Throwable be instantiated?

**Yes.**

```
Throwable t =
new Throwable();
```

Although it is rarely done in practice.

---

## 4. Can we throw Throwable?

**Yes.**

```
throw new Throwable();
```

Because it is a checked exception, you must catch it or declare it with `throws`.

---

## 5. Can we catch Throwable?

**Yes.**

But it is generally discouraged because it catches both exceptions and errors.

---

## 6. Which methods come from Throwable?

- `getMessage()`
- `printStackTrace()`
- `getCause()`
- `getStackTrace()`
- `toString()`
- `getSuppressed()`
- `initCause()`

---

## 7. Where is Throwable stored?

Like all Java objects, a `Throwable` object is created on the **heap**.

---

## 8. Why does Exception extend Throwable instead of Object?

Because only subclasses of `Throwable` are allowed to participate in Java's exception handling mechanism (`throw`, `throws`, and `catch`).

---

# Common Interview Mistakes

❌ **Mistake 1**

> "Throwable and Exception are the same."

**Correct:** `Exception` is a subclass of `Throwable`.

---

❌ **Mistake 2**

> "Throwable should always be caught."

**Correct:** Usually catch `Exception`. Catching `Throwable` may also catch serious JVM `Error`s that are typically not recoverable.

---

❌ **Mistake 3**

> "`Throwable` is an interface."

**Correct:** It is a **class**.

---

```
             Object

          /         \
```

### Key Points

- Root class of Java exception hierarchy.
- Parent of both `Exception` and `Error`.
- Provides methods like `getMessage()`, `printStackTrace()`, and `getCause()`.
- Any object used with `throw` or `catch` must extend `Throwable`.
- Avoid catching `Throwable` in normal application code because it also catches serious `Error`s.
