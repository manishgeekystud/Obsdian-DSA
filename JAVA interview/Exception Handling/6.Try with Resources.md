
# What is Try-With-Resources?

**Definition:**

Try-With-Resources is a special form of the `try` statement that **automatically closes resources** after they are no longer needed.

Before Java 7, developers had to close resources manually using the `finally` block.

Java 7 introduced Try-With-Resources to eliminate this boilerplate code.

---

# Why Do We Need Try-With-Resources?

Consider reading a file.

## Before Java 7

```
import java.io.*;

public class Demo {

    public static void main(String[] args) {

        BufferedReader br = null;

        try {

            br = new BufferedReader(new FileReader("abc.txt"));

            System.out.println(br.readLine());

        }

        catch(IOException e){

            e.printStackTrace();

        }

        finally{

            try{

                if(br != null)
                    br.close();

            }

            catch(IOException e){

                e.printStackTrace();

            }

        }

    }

}
```

### Problems

- Large amount of code
- Nested `try-catch`
- Easy to forget `close()`
- Risk of resource leaks

---

# Java 7 Solution

```
import java.io.*;

public class Demo {

    public static void main(String[] args) {

        try (BufferedReader br =
                new BufferedReader(new FileReader("abc.txt"))) {

            System.out.println(br.readLine());

        }

        catch(IOException e){

            e.printStackTrace();

        }

    }

}
```

No `finally` block is required.

The JVM automatically closes `BufferedReader`.

---

# Syntax

```
try(Resource declaration){

    // use resource

}
catch(Exception e){

}
```

Example

```
try(FileInputStream fis =
        new FileInputStream("abc.txt")){

}
```

# How Does It Work?

```

   /        \

  |          |

   \        /

       |

```

The resource is closed automatically after the `try` block finishes.

# AutoCloseable Interface

```
public interface AutoCloseable {

    void close() throws Exception;

}
```

Any class implementing this interface can be used inside Try-With-Resources.

---

# Example

```
Scanner sc = new Scanner(System.in);

try(sc){

    System.out.println(sc.nextInt());

}
```

`Scanner` implements `Closeable`, which extends `AutoCloseable`.

---

# Multiple Resources

You can declare multiple resources.

```
try(

    FileInputStream fis =
        new FileInputStream("input.txt");

    FileOutputStream fos =
        new FileOutputStream("output.txt")

){

}
```

Resources are separated by semicolons.

---

# Closing Order (Very Important)

Resources are closed in **reverse order** of creation.

Example

```
try(

    Resource1 r1 = new Resource1();
    Resource2 r2 = new Resource2();
    Resource3 r3 = new Resource3()

){

}
```

Closing order

```
Create

r1

r2

r3

Close

r3

r2

r1
```

This is called **LIFO (Last In, First Out)**.

---

# Dry Run

```
try(
```

Execution

|Step|Action|

# Exception During Resource Closing

Suppose

```
try(Resource r = new Resource()){

    throw new RuntimeException();

}
```

If:

- try throws exception
- close() also throws exception

Which exception is preserved?

Answer:

The exception from **try block**.

The exception from `close()` becomes a **Suppressed Exception**.

---

# Suppressed Exception

Example

```
try(Resource r = new Resource()){

    throw new Exception("Main Exception");

}
```

Suppose close()

```
close(){

    throw new Exception("Close Exception");

}
```

Output

```
Main Exception

Suppressed:
Close Exception
```

Retrieve them

```
catch(Exception e){

    for(Throwable t : e.getSuppressed()){

        System.out.println(t);

    }

}
```

# Custom Resource

Any custom class implementing `AutoCloseable` can be used.

```
class MyResource implements AutoCloseable {

    public void close(){

        System.out.println("Closed");

    }

}
```

Usage

```
try(MyResource r = new MyResource()){

    System.out.println("Working");

}
```

Output

```
Working

Closed
```

# Try-With-Resources vs Finally

|Feature|Finally|Try-With-Resources|
|---|---|---|
|Automatic closing|❌ No|✅ Yes|
|Boilerplate code|High|Low|
|Nested try required|Usually|No|
|Cleaner code|❌|✅|
|Java Version|All|Java 7+|

# Java 9 Enhancement

Before Java 9

```
BufferedReader br =
new BufferedReader(new FileReader("abc.txt"));

try(br){

}
```

❌ Compilation Error (Java 7/8)

Need

```
try(BufferedReader br =
new BufferedReader(...)){

}
```

---

### Java 9+

Now allowed

```
BufferedReader br =
new BufferedReader(new FileReader("abc.txt"));

try(br){

    System.out.println(br.readLine());

}
```

The resource can be declared before the `try` block as long as it is **effectively final** (not reassigned).

# Common Mistakes

## Mistake 1

Using a class that doesn't implement `AutoCloseable`

```
class Employee{

}

try(Employee e = new Employee()){

}
```

Compilation Error

Reason

Employee doesn't implement `AutoCloseable`.

---

## Mistake 2

Manually closing inside try

```
try(FileInputStream fis =
new FileInputStream("abc.txt")){

    fis.close();

}
```

Not needed.

It closes automatically.

---

## Mistake 3

Using resource after try

```
FileInputStream fis;

try(fis =
new FileInputStream("abc.txt")){

}

fis.read();
```

The resource has already been closed.

---

# Best Practices

✔ Use Try-With-Resources for every closable resource.

✔ Prefer it over `finally` for resource cleanup.

✔ Do not manually call `close()` inside the `try` block.

✔ Keep the resource declaration simple and readable.

✔ Use multiple resources only when necessary.

# Frequently Asked Interview Questions

## 1. What is Try-With-Resources?

A special `try` statement that automatically closes resources implementing `AutoCloseable`.

---

## 2. Which Java version introduced it?

**Java 7**

---

## 3. Which interface is required?

**AutoCloseable**

---

## 4. Can multiple resources be declared?

**Yes.**

```
try(
    Scanner sc = new Scanner(System.in);
    BufferedReader br =
        new BufferedReader(new FileReader("abc.txt"))
){

}
```

---

## 5. In which order are resources closed?

**Reverse order (LIFO).**

---

## 6. What if both the try block and close() throw exceptions?

The exception from the **try block** is propagated.

The exception from `close()` is stored as a **suppressed exception**.

---

## 7. Is finally still required?

Not for closing resources managed by Try-With-Resources.

You can still use a `finally` block for other cleanup tasks if needed.

---

## 8. Can we use our own class?

Yes, if it implements `AutoCloseable`.

---

## 9. What changed in Java 9?

Resources declared before the `try` statement can be reused if they are **effectively final**.

# Interview Tip

A common interview question is:

> **Why is Try-With-Resources better than using `finally`?**

A strong answer is:

> **Try-With-Resources automatically closes resources, reduces boilerplate code, prevents resource leaks, correctly handles suppressed exceptions, and is less error-prone than manually closing resources in a `finally` block.**
