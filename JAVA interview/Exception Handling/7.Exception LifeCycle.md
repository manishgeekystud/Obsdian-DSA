**Exception Lifecycle** is the complete journey of an exception from the moment an error occurs until it is handled or the program terminates.

In simple words:

> **Exception Lifecycle = Creation → Throwing → Propagation → Handling → Program Continues (or JVM Terminates)**

This is one of the most commonly asked interview topics because it explains **how the JVM internally handles exceptions**.

# Exception Lifecycle Diagram

```

                    |

             /               \
           Found            Not Found
             |                 |
             V                 V
     5. Handle Exception   Propagate to Caller
             |                 |
             |            Search Again
             |                 |
             +--------+--------+
                      |
          Handler Found?
             /          \
           Yes          No
            |            |
            V            V
 Program Continues   JVM Default Handler
                           |
                           V
                    Program Terminates
```

"The Exception Lifecycle in Java is the complete process from the occurrence of an exception until it is either handled or the program terminates.

First, while executing the program, if the JVM encounters an abnormal condition, such as dividing a number by zero or accessing a null object, it creates an appropriate exception object. This object is created on the heap and contains information like the exception type, message, and stack trace.

Next, the JVM throws that exception. As soon as the exception is thrown, the normal flow of the current method stops, and the remaining statements in that method are skipped.

Then, the JVM searches for a matching catch block in the current method. If a matching catch block is found, it executes that block, and after that the program continues normally.

If no matching catch block is found, the exception propagates to the calling method. This process continues up the call stack until a suitable handler is found.

During this propagation, if any method has a finally block, it executes before the method exits.

If the exception reaches the main method and is still not handled, the JVM's default exception handler prints the stack trace and terminates the application."

# Explain with a Small Example (Interview Style)

```
class Demo {

    static void m3() {
        int x = 10 / 0;
    }

    static void m2() {
        m3();
    }

    static void m1() {
        m2();
    }

    public static void main(String[] args) {
        try {
            m1();
        } catch (ArithmeticException e) {
            System.out.println("Handled");
        }
    }
}
```

Then explain:

> "Here, the exception occurs inside `m3()`. The JVM creates an `ArithmeticException` object and throws it. Since `m3()` doesn't handle it, it propagates to `m2()`, then to `m1()`, and finally to `main()`, where a matching catch block is found. After executing the catch block, the program continues."

# If the Interviewer Asks: "Who Creates the Exception Object?"

**Answer:**

> "For built-in exceptions like `NullPointerException` or `ArithmeticException`, the JVM creates the exception object automatically. For custom exceptions, the developer explicitly creates the object using the `new` keyword and throws it using the `throw` statement."

Example:

```
throw new InvalidAgeException("Age must be greater than 18");
```

---

# If the Interviewer Asks: "Where is the Exception Object Stored?"

**Answer:**

> "The exception object is created on the Heap because every Java object is allocated on the heap. The method call information is maintained in the Stack, which is why stack unwinding happens during exception propagation."

---

# If the Interviewer Asks: "Does finally Execute During Exception Propagation?"

**Answer:**

> "Yes. Before an exception leaves a method and propagates to its caller, the JVM executes the `finally` block, unless the JVM terminates abruptly, such as through `System.exit(0)`."

---

# 30-Second Answer (When the Interviewer Wants a Short Response)

> "The Exception Lifecycle starts when an exception occurs. The JVM creates an exception object, throws it, and searches for a matching catch block in the current method. If no handler is found, the exception propagates up the call stack. During propagation, any finally blocks execute before the method exits. If no handler is found anywhere, the JVM's default exception handler prints the stack trace and terminates the program."
