# Q1. What are Multiple Catch Blocks?

### Answer

Multiple catch blocks allow a single `try` block to handle **different types of exceptions separately**.

Instead of writing one generic catch block, we can write multiple catch blocks, each handling a specific exception.

Example:

```
public class Demo {

    public static void main(String[] args) {

        try {

            String s = null;
            System.out.println(s.length());

        }

        catch (ArithmeticException e) {

            System.out.println("Arithmetic Exception");

        }

        catch (NullPointerException e) {

            System.out.println("Null Pointer Exception");

        }

    }
}
```

Output

```
Null Pointer Exception
```

# Q2. Why do we use Multiple Catch Blocks?

### Answer

Different exceptions require different handling.

Example:

```
try {

    FileReader fr = new FileReader("abc.txt");

    int x = 10 / 0;

}

catch(FileNotFoundException e){

    System.out.println("File Missing");

}

catch(ArithmeticException e){

    System.out.println("Divide by Zero");

}
```

Each exception has its own recovery logic.

# Q4. Which catch block executes?

### Answer

Only **one matching catch block** executes.

Once a matching catch block is found:

- Remaining catch blocks are skipped.
- Control moves to `finally` (if present).

Example

```
try{

    int x = 10 / 0;

}

catch(ArithmeticException e){

    System.out.println("Arithmetic");

}

catch(Exception e){

    System.out.println("Exception");

}
```

Output

```
Arithmetic
```

Only the first matching catch executes.

---

# Q5. What happens if no catch block matches?

### Example

```
try{

    int x = 10 / 0;

}

catch(NullPointerException e){

    System.out.println("NPE");

}
```

Output

```
Exception in thread "main"
ArithmeticException
```

### Answer

The JVM propagates the exception.

If nobody handles it, the program terminates.

# Q6. Why should child exceptions come before parent exceptions?

### Correct

```
try{

}
catch(FileNotFoundException e){

}
catch(IOException e){

}
```

---

### Wrong

```
try{

}
catch(IOException e){

}
catch(FileNotFoundException e){

}
```

Compilation Error

```
Unreachable catch block
```

### Explanation

```
IOException
      |
FileNotFoundException
```

If `IOException` is caught first, it already catches every `FileNotFoundException`.

Therefore, the second catch block can never execute.

# Q7. What is an Unreachable Catch Block?

Answer:

A catch block that can **never execute** because a parent exception has already handled it.

Example

```
try{

}
catch(Exception e){

}
catch(ArithmeticException e){

}
```

Compilation Error

```
Unreachable catch block
```

# Q10. What happens if an exception occurs inside a catch block?

Example

```
try{

    int x = 10 / 0;

}

catch(ArithmeticException e){

    String s = null;

    System.out.println(s.length());

}
```

Output

```
NullPointerException
```

### Answer

The new exception propagates unless handled by another enclosing `try-catch`.

### Answer


# Most Asked Interview Coding Question

### Predict the Output

```
public class Test {

    public static void main(String[] args) {

        try {

            String s = null;

            System.out.println(s.length());

        }

        catch(RuntimeException e){

            System.out.println("Runtime");

        }

        catch(Exception e){

            System.out.println("Exception");

        }

    }

}
```

### Output

```
Runtime
```

### Explanation

Hierarchy:

```
Exception
     |
RuntimeException
     |
NullPointerException
```

`NullPointerException` is a subclass of `RuntimeException`, so the **first matching catch block** (`RuntimeException`) handles it. The later `Exception` catch block is skipped.

---

# Interview Tip

Remember these three golden rules:

1. **The JVM executes only the first matching `catch` block.**
2. **Always catch child exceptions before parent exceptions.**
3. **If no `catch` block matches, the exception propagates to the caller (or terminates the program if unhandled).**
