it explains **how an exception travels through the method call stack until it is handled.**

# What is Exception Propagation?

**Definition:**

Exception Propagation is the process in which an exception **moves from the method where it occurred to its caller**, and then to the caller's caller, until:

- A matching `catch` block is found, or
- The exception reaches the JVM, which terminates the program.

In simple words:

> **When a method cannot handle an exception, it passes (propagates) the exception to the method that called it.**

---

# Real-Life Analogy

Imagine a company hierarchy.

```
Employee
    |
Team Lead
    |
Manager
    |
Director
```

Suppose an employee has a problem.

- Employee can't solve it → asks Team Lead.
- Team Lead can't solve it → asks Manager.
- Manager can't solve it → asks Director.

Similarly:

```
method3()
      |
method2()
      |
method1()
      |
main()
      |
JVM
```

If one method cannot handle the exception, it propagates upward.

# Example 1: Exception Propagation

```
public class Demo {

    static void method3() {

        int x = 10 / 0;

    }

    static void method2() {

        method3();

    }

    static void method1() {

        method2();

    }

    public static void main(String[] args) {

        method1();

    }

}
```

### Output

```
Exception in thread "main"
java.lang.ArithmeticException: / by zero
```

# Example 2: Catch in Caller

```
public class Demo {

    static void method3() {

        int x = 10 / 0;

    }

    static void method2() {

        method3();

    }

    static void method1() {

        try {

            method2();

        }

        catch (ArithmeticException e) {

            System.out.println("Handled");

        }

    }

    public static void main(String[] args) {

        method1();

    }

}
```

### Output

```
Handled
```

# Stack Unwinding

When an exception propagates, the JVM removes stack frames one by one.

Suppose

```
main()
{
    method1();
}

method1()
{
    method2();
}

method2()
{
    method3();
}

method3()
{
    int x = 10/0;
}
```

Call Stack

```
Top

method3()

method2()

method1()

main()

Bottom
```

After exception

```
method3() removed

method2() removed

method1() removed

main()
```

This process is called **Stack Unwinding**.

# Checked Exception Propagation

Checked exceptions must be declared using **throws**.

Example

```
import java.io.*;

class Demo {

    static void readFile() throws IOException {

        FileReader fr = new FileReader("abc.txt");

    }

    static void process() throws IOException {

        readFile();

    }

    public static void main(String[] args) {

        try {

            process();

        }

        catch(IOException e){

            System.out.println("Handled");

        }

    }

}
```

Propagation

```
readFile()

      |

process()

      |

main()

      |

catch(IOException)
```

---

# Runtime Exception Propagation

Runtime exceptions propagate automatically.

Example

```
static void m3() {

    String s = null;

    s.length();

}
```

No **throws** keyword is required.

---

# Checked vs Runtime Propagation

|Feature|Checked Exception|Runtime Exception|
|---|---|---|
|Compiler checks|Yes|No|
|`throws` required|Yes (unless handled)|No|
|Propagates automatically at runtime|Yes|Yes|
|Compile-time declaration|Required|Not required|

> **Important:** Both checked and unchecked exceptions **propagate at runtime**. The difference is that checked exceptions must also satisfy the compiler by being caught or declared with `throws`.

---

# Propagation with throws

```
class Demo {

    static void m3() throws Exception {

        throw new Exception();

    }

    static void m2() throws Exception {

        m3();

    }

    static void m1() throws Exception {

        m2();

    }

    public static void main(String[] args) {

        try {

            m1();

        }

        catch(Exception e){

        }

    }

}
```

Flow

```
m3()

↓

m2()

↓

m1()

↓

main()

catch
```

# Can finally Execute During Propagation?

Yes.

```
static void m3() {

    try {

        int x = 10 / 0;

    }

    finally {

        System.out.println("Finally");

    }

}
```

Output

```
Finally

Exception...
```

Even though the exception propagates, `finally` executes before leaving the method.

---

# Can Error Propagate?

Yes.

Example

```
public class Demo {

    public static void main(String[] args) {

        main(args);

    }

}
```

Eventually

```
StackOverflowError
```

The `Error` also propagates up the call stack until the JVM handles it.

# Exception Propagation Rules

### Rule 1

Exceptions propagate **upward**, never downward.

```
method3()

↓

method2()

↓

method1()

↓

main()
```

---

### Rule 2

Propagation stops when a matching `catch` block is found.

---

### Rule 3

If nobody catches the exception, the JVM handles it.

---

### Rule 4

Checked exceptions require either:

- `throws`
- `try-catch`

---

### Rule 5

Runtime exceptions propagate automatically.

---

# Common Mistakes

### ❌ Thinking propagation works downward

Wrong

```
main()

↓

method1()
```

Correct

```
method3()

↑

method2()

↑

method1()

↑

main()
```

Propagation follows the **reverse direction of method calls**.

---

### ❌ Believing only checked exceptions propagate

Wrong.

Both checked and runtime exceptions propagate.

---

### ❌ Forgetting `throws`

```
void read() {

    FileReader fr = new FileReader("abc.txt");

}
```

Compilation error because `IOException` is checked and neither handled nor declared.

---

# Best Practices

- Handle exceptions at the **appropriate layer**, not necessarily where they occur.
- Use `throws` to propagate checked exceptions when the current method cannot recover.
- Catch exceptions only when you can add meaningful handling or recovery.
- Avoid catching `Exception` everywhere, as it hides the real problem.

---

# Interview Questions

### 1. What is Exception Propagation?

It is the process of passing an exception from the current method to its caller until it is handled or reaches the JVM.

---

### 2. Which exceptions propagate?

**Answer:** Both checked and unchecked (runtime) exceptions propagate at runtime.

---

### 3. Does propagation occur for checked exceptions?

**Yes.**

However, checked exceptions must also be declared with `throws` (or caught) to satisfy the compiler.

---

### 4. Does propagation occur for runtime exceptions?

**Yes.**

They propagate automatically without requiring `throws`.

---

### 5. What is Stack Unwinding?

When an exception propagates, the JVM removes method stack frames one by one until it finds a matching `catch` block or reaches the top of the call stack.

---

### 6. Can finally execute during propagation?

**Yes.**

The `finally` block executes before the exception leaves the method.

---

### 7. Does propagation stop after a catch block?

**Yes.**

Once a matching `catch` block handles the exception, normal execution resumes after the `try-catch-finally` statement (unless another exception is thrown).

## Key Points

- Exception propagation moves **up the call stack**.
- Propagation continues until a matching `catch` block is found.
- If no handler exists, the **JVM handles the exception** and terminates the program.
- `finally` executes **before** the exception leaves a method.
- Checked exceptions require `throws` (or `try-catch`) in addition to runtime propagation; runtime exceptions propagate without compiler enforcement.
