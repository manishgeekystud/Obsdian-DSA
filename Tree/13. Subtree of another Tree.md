# Subtree of Another Tree (Subtree Problem)

> **Interview Frequency:** ⭐⭐⭐⭐⭐ (Very Frequently Asked)

This problem is asked in **Amazon, Walmart, Microsoft, Google, Oracle, Adobe, Flipkart, Zoho, TCS Digital** and many other product companies.

---

# Problem Statement

Given the roots of two binary trees:

- `root1` → Main Tree
- `root2` → Sub Tree

Determine whether `root2` is a **subtree** of `root1`.

A subtree is a node in `root1` along with **all of its descendants**.

Return:

- `true` → if `root2` exists as a subtree.
- `false` → otherwise.

---

# Example 1

## Main Tree

```
            10
          /    \
         20     30
        /  \
      40    50
```

## Sub Tree

```
        20
       /  \
     40    50
```

Output

```
true
```

Reason

Node **20** together with all its descendants forms the second tree.

---

# Example 2

## Main Tree

```
            10
          /    \
         20     30
        /  \
      40    50
```

## Sub Tree

```
       20
      /
    40
```

Output

```
false
```

Reason

The subtree rooted at **20** in the main tree also contains **50**.

Both trees must be **identical**, not partially identical.

# Observation

We need to answer two questions:

### Question 1

Is the current node the root of the subtree?

If yes

↓

Compare both trees.

---

### Question 2

If not,

Search

- Left subtree
- Right subtree

---

# Idea

Traverse the main tree.

Whenever

```
root1.data == root2.data
```

check

```
isIdentical(root1, root2)
```

If identical

↓

Return

```
true
```

Else

↓

Continue searching.

---

# Algorithm

```
isSubTree(root1, root2)

1. If both trees are NULL

      return true

2. If one tree is NULL

      return false

3. If current node values are equal

       Check if both trees are identical

       If identical

             return true

4. Search in left subtree

5. Search in right subtree

6. Return

left OR right
```

# Java Code (Interview Standard)

```java
class Solution {

    public boolean isSubTree(Node root1, Node root2) {

        // If subtree is empty, it is always a subtree
        if (root2 == null)
            return true;

        // Main tree finished but subtree still exists
        if (root1 == null)
            return false;
        // Current node values match
        if (root1.data == root2.data) {

            // Check whether both trees are identical
            if (isIdentical(root1, root2))
                return true;
        }
        // Continue searching
        return isSubTree(root1.left, root2)
            || isSubTree(root1.right, root2);
    }

    static boolean isIdentical(Node r1, Node r2) {

        // Both nodes are null
        if (r1 == null && r2 == null)
            return true;

        // One node is null
        if (r1 == null || r2 == null)
            return false;

        // Data mismatch
        if (r1.data != r2.data)
            return false;

        // Compare left and right subtrees
        return isIdentical(r1.left, r2.left)
            && isIdentical(r1.right, r2.right);
    }
}
```

# Why We Don't Return Immediately

Suppose

```
          10
         /  \
        5    5
            /
           8
```

Subtree

```
      5
     /
    8
```

There are **two nodes with value 5**.

First one

↓

Not identical.

Second one

↓

Identical.

If we write

```
if(root1.data == root2.data)
    return isIdentical(root1, root2);
```

then the function returns **false** after checking the **first 5**, never reaching the second.

---

# Correct Way

```
if(root1.data == root2.data) {

    if(isIdentical(root1, root2))
        return true;
}
```

If not identical,

continue searching.

---

# Time Complexity

Let:

- `N` = Number of nodes in `root1`
- `M` = Number of nodes in `root2`

Worst case

```
O(N × M)
```

Reason:

For every node in `root1`, we may compare up to all nodes in `root2`.

---

# Space Complexity

Recursive stack

Balanced Tree

```
O(log N)
```

Worst case (skewed)

```
O(N)
```

# Common Mistakes

### ❌ Mistake 1

```
return isIdentical(root1, root2);
```

This stops searching after the **first matching value**.

---

### ❌ Mistake 2

Checking

```
root1.left != null &&
root1.right != null
```

before calling `isIdentical()`.

A subtree can start at **any node**, regardless of whether it has zero, one, or two children.

---

### ❌ Mistake 3

Comparing only node values.

```
root1.data == root2.data
```

Matching values do **not** guarantee matching subtrees.

Always verify with `isIdentical()`.

---

# Interview Questions

### 1. Why do we need `isIdentical()`?

Because matching node values alone do not ensure the entire subtree matches. We must compare both structure and data.

---

### 2. Why don't we return immediately when the values match?

There may be multiple nodes with the same value in the main tree. If the first match fails, another matching node might contain the correct subtree.

---

### 3. Can this be optimized?

Yes. A common optimization is to serialize both trees (including null markers) and use a string matching algorithm such as **KMP** or **Rabin–Karp**, reducing the average matching cost. Another approach uses tree hashing.

---

### 4. What is the relationship between `isSubTree()` and `isIdentical()`?

- `isSubTree()` searches **where** the subtree could begin.
- `isIdentical()` verifies **whether** the subtree rooted at that node is exactly the same.

---

# Interview Tip ⭐⭐⭐⭐⭐

Think of this problem as **two recursive algorithms working together**:

```
               isSubTree()
                    │
     Searches every node in root1
                    │
     ┌──────────────┴──────────────┐
     │                             │
Values don't match          Values match
     │                             │
Continue searching         Call isIdentical()
                                   │
                        ┌──────────┴──────────┐
                        │                     │
                    Identical             Not identical
                        │                     │
                  Return true         Continue searching
```

## Pattern to Remember

Many tree interview problems use this combination:

- **Traversal recursion** → to find the candidate node.
- **Verification recursion** → to validate the subtree.

Once you recognize this pattern, problems like **Subtree of Another Tree**, **Identical Tree**, **Symmetric Tree**, and **Flip Equivalent Binary Trees** become much easier to solve.
