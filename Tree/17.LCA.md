# Lowest Common Ancestor (LCA) of a Binary Tree – Complete Interview Notes

The **Lowest Common Ancestor (LCA)** is one of the **Top 10 Binary Tree interview questions**. It is asked in Amazon, Microsoft, Google, Adobe, Oracle, Walmart, Flipkart, Samsung, and many product-based companies.

---

# Problem Statement

Given:

- Root of a Binary Tree
- Two nodes `n1` and `n2`

Find their **Lowest Common Ancestor (LCA).**

---

# What is Lowest Common Ancestor?

The **Lowest Common Ancestor** is the **lowest node** in the tree that has **both nodes as descendants** (a node can be a descendant of itself).

# Approach 1 – Store Root-to-Node Paths (Brute Force)

## Idea

1. Find path from Root to Node1.
2. Find path from Root to Node2.
3. Compare both paths.
4. Last common node is LCA.

## Code

```java
class Solution {

    Node lca(Node root, int n1, int n2) {

        ArrayList<Node> path1 = new ArrayList<>();
        ArrayList<Node> path2 = new ArrayList<>();

        getPath(root, n1, path1);
        getPath(root, n2, path2);

        int i = 0;

        while (i < path1.size()
                && i < path2.size()
                && path1.get(i) == path2.get(i)) {
            i++;
        }

        return path1.get(i - 1);
    }

    boolean getPath(Node root,
                    int key,
                    ArrayList<Node> path) {

        if (root == null)
            return false;

        path.add(root);

        if (root.data == key)
            return true;

        if (getPath(root.left, key, path)
                || getPath(root.right, key, path))
            return true;

        path.remove(path.size() - 1);

        return false;
    }
}
```

# Java Code

```java
class Solution {

    Node lca(Node root, int n1, int n2) {

        if (root == null)
            return null;

        if (root.data == n1 || root.data == n2)
            return root;

        Node left = lca(root.left, n1, n2);

        Node right = lca(root.right, n1, n2);

        if (left != null && right != null)
            return root;

        if (left != null)
            return left;

        return right;
    }
}
```

# Why Does This Work?

Every recursive call answers one question:

> **Does my subtree contain either of the target nodes?**

- If neither exists → return `null`
- If exactly one exists → return that node
- If one is found in each subtree → current node is the LCA

---

# Common Mistakes

### Mistake 1

Thinking LCA always lies above both nodes.

Wrong.

```
LCA(2,4)

Answer

2
```

---

### Mistake 2

Returning current node immediately.

Current node is returned **only when**:

```
Left != NULL

AND

Right != NULL
```

or it matches one of the target nodes.

---

### Mistake 3

Forgetting the base case.

```
if(root==null)
    return null;
```

---

### Mistake 4

Confusing Binary Tree with BST.

For a **BST**, LCA can be found using node values in **O(H)** without searching both subtrees.

For a **general Binary Tree**, you must search both sides.

---

# Binary Tree vs BST LCA

|Binary Tree|Binary Search Tree|
|---|---|
|Search both subtrees|Use BST property|
|Time: O(N)|Time: O(H)|
|Recursive DFS|Value comparisons|

---

# Interview Questions

### 1. Why do we return `root` when it matches one of the target nodes?

Because a node can be the ancestor of itself.

---

### 2. Why does `left != null && right != null` mean the current node is the LCA?

It means one target node was found in the left subtree and the other in the right subtree. The first node where this happens is their lowest common ancestor.

---

### 3. Why don't we continue searching after finding the LCA?

The recursive returns propagate the answer upward. Once an LCA is determined in a subtree, ancestor calls simply pass it back.

---

### 4. Why is the optimized approach O(N)?

Each node is visited at most once, and only constant work is performed at each node.
