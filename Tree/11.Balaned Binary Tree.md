This is one of the most common Binary Tree interview questions asked in **Amazon, Microsoft, Google, Oracle, Adobe, Walmart, Flipkart, TCS Digital**, etc.

# Problem Statement

A Binary Tree is called **Balanced** if, for **every node**, the difference between the heights of its left and right subtrees is **not more than 1**.

Mathematically,

```
| Height(Left Subtree) - Height(Right Subtree) | <= 1
```

This condition must hold **for every node** in the tree, not just the root.

```

         2     3
        /
       4
      /

At root:

```

Already unbalanced.

But even if the root were balanced, **every node** must still satisfy the condition.

# Brute Force Approach

For every node:

1. Calculate left height.
2. Calculate right height.
3. Check difference.
4. Repeat for left subtree.
5. Repeat for right subtree.

---

## Algorithm

```
isBalanced(root)

1. If root is NULL
      return true

2. leftHeight = height(root.left)

3. rightHeight = height(root.right)

4. if |leftHeight-rightHeight| >1
      return false

5. return

isBalanced(left)

AND

isBalanced(right)
```

---

# Java Code (Brute Force)

```
class Node {

    int data;
    Node left, right;

    Node(int data) {
        this.data = data;
    }
}

public class BalancedTree {

    static int height(Node root) {

        if (root == null)
            return 0;

        return 1 + Math.max(height(root.left),
                            height(root.right));
    }

    static boolean isBalanced(Node root) {

        if (root == null)
            return true;

        int leftHeight = height(root.left);

        int rightHeight = height(root.right);

        if (Math.abs(leftHeight - rightHeight) > 1)
            return false;

        return isBalanced(root.left)
                && isBalanced(root.right);
    }
}
```

# Why is this Slow?

Suppose

```
            1
          /   \
         2     3
        / \
       4   5
```

For root

```
height(2)

height(3)
```

are calculated.

Then

```
isBalanced(2)
```

again calls

```
height(4)

height(5)
```

Many heights are computed repeatedly.

---

# Time Complexity

Height takes

```
O(n)
```

and is called for every node.

Overall

```
O(n²)
```

This is **not preferred** in interviews.

---

# Optimized Approach (Most Asked)

Instead of computing height separately,

**calculate height and balance together in one recursion.**

---

# Idea

Each recursive call returns:

- Height of subtree if balanced.
- `-1` if subtree is unbalanced.

If any subtree returns `-1`, stop immediately.

---

# Algorithm

```
height(root)

If root == NULL

    return 0

left = height(left)

If left == -1

    return -1

right = height(right)

If right == -1

    return -1

If abs(left-right)>1

    return -1

Return

1 + max(left,right)
```

---

# Java Code (Optimized)

```
class Node {

    int data;
    Node left, right;

    Node(int data) {
        this.data = data;
    }
}

public class BalancedBinaryTree {

    static int checkHeight(Node root) {

        // Empty tree
        if (root == null)
            return 0;

        // Height of left subtree
        int left = checkHeight(root.left);

        // Left subtree is unbalanced
        if (left == -1)
            return -1;

        // Height of right subtree
        int right = checkHeight(root.right);

        // Right subtree is unbalanced
        if (right == -1)
            return -1;

        // Current node is unbalanced
        if (Math.abs(left - right) > 1)
            return -1;

        // Return height
        return 1 + Math.max(left, right);
    }

    static boolean isBalanced(Node root) {
        return checkHeight(root) != -1;
    }
}
```

# Why Return `-1`?

`-1` acts as a **special signal** meaning:

```
"This subtree is already unbalanced."
```

Once a subtree returns `-1`, there is no need to continue calculating heights for higher nodes. This avoids unnecessary work and keeps the algorithm efficient.

---

# Time Complexity Comparison

|Approach|Time|Space|
|---|---|---|
|Brute Force|O(n²)|O(h)|
|Optimized|O(n)|O(h)|

where `h` is the height of the tree.

---

# Interview Questions

### 1. Why is the brute-force approach `O(n²)`?

Because the `height()` function is called repeatedly for the same subtrees while checking every node.

---

### 2. Why is the optimized approach `O(n)`?

Each node is visited **exactly once**, and height and balance are computed together during the same traversal.

---

### 3. Why use `-1`?

A valid height is never negative, so `-1` is a convenient sentinel value indicating that the subtree is unbalanced.

---

### 4. Why check the left subtree before the right subtree?

If the left subtree is already unbalanced (`-1`), we can return immediately without doing more work. This is an example of **early termination**.

---

# Interview Tip

Many binary tree interview questions follow a **postorder traversal** pattern (Left → Right → Current), where information from the children is used to compute the result for the parent.

Examples:

|Problem|Information Returned|
|---|---|
|Height|Height|
|Size|Number of nodes|
|Maximum|Maximum value|
|Balanced Binary Tree|Height or `-1` if unbalanced|
|Diameter (Optimized)|Height while updating diameter|

Recognizing this **postorder recursion pattern** is one of the most valuable skills for solving binary tree interview problems efficiently.
