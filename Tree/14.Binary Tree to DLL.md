Given the **root** of a binary tree, convert it to a Doubly Linked List (DLL) in place, using the same node structure.

- The left and right pointers in the binary tree nodes should be used as prev and next pointers respectively in the resulting DLL.
- The DLL should be formed by performing an inorder traversal of the binary tree (i.e., Left -> Root -> Right).
- The first node in the inorder traversal (i.e., the leftmost node) should become the head of the DLL.

Return the head of the resulting DLL.

**Examples:**

**Input:** root **=** [1, 2, 3]
**Output:** 2 1 3  
3 1 2  
**Explanation:** Inorder traversal visits 2, 1, 3. Node 2 becomes the head of the DLL, giving the list 2 <=> 1 <=> 3.![](https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/700144/Web/Other/blobid3_1768475771.jpg)  

**Input:** root = [10, 20, 30, 40, 60]
**Output:**   
40 20 60 10 30 
30 10 60 20 40 
**Explanation:** Inorder traversal visits 40, 20, 60, 10, 30. Node 40 becomes the head of the DLL, giving the list 40 <=> 20 <=> 60 <=> 10 <=> 30_._  
![](https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/700144/Web/Other/blobid2_1763203489.webp)

**Constraints:**  
1 ≤ number of nodes ≤ 106  
0 ≤ node->data ≤ 105

# Problem Statement

Convert a **Binary Tree** into a **Doubly Linked List (DLL)** **in-place**.

The conversion should follow **Inorder Traversal**.

Use:

- `left` pointer as **previous**
- `right` pointer as **next**

Do **not** create any new nodes.

---

## Given Node Structure

```
class Node {
    int data;
    Node left, right;

    Node(int x) {
        data = x;
        left = right = null;
    }
}
```

---

# Example

## Binary Tree

```
           10
         /    \
       20      30
      /  \    /
    40   50 60
```

---

## Inorder Traversal

```
40 20 50 10 60 30
```

---

## Expected DLL

```
NULL <- 40 <-> 20 <-> 50 <-> 10 <-> 60 <-> 30 -> NULL
```

Notice

```
left  = previous
right = next
```

---

# Observation

The required DLL order is exactly the **Inorder Traversal**.

So the question becomes

> During Inorder Traversal, connect every current node with the previously visited node.

This is the entire idea.

---

# Key Idea

Maintain two pointers

```
prev
head
```

### prev

Points to the previously visited node during inorder traversal.

### head

Points to the first node of the DLL.

The first inorder node becomes the head.

---

# Code

```java
class Solution {

    Node prev = null;

    public Node treeToDLL(Node root) {

        if (root == null)
            return root;

        Node head = treeToDLL(root.left);

        if (prev == null)
            head = root;
        else {
            root.left = prev;
            prev.right = root;
        }

        prev = root;

        treeToDLL(root.right);

        return head;
    }
}
```

# Why Does It Work?

Because inorder traversal visits nodes in the exact order required for the DLL. At each visit, `prev` always points to the immediately preceding node in that order, so linking `prev` and the current node builds the list correctly.

---

# Complexity Analysis

### Time Complexity

Every node is visited once.

```
O(N)
```

---

### Space Complexity

Only recursion stack is used.

```
O(H)
```

where `H` is the tree height.

- Balanced tree: `O(log N)`
- Skewed tree: `O(N)`

---

# Common Mistakes

### Mistake 1

Forget

```
prev = root;
```

Then all nodes try to connect with the same previous node.

---

### Mistake 2

Write

```
root.right = prev;
```

Wrong direction.

Correct

```
prev.right = root;
```

---

### Mistake 3

Forget

```
root.left = prev;
```

Then backward traversal won't work.

---

### Mistake 4

Declare `prev` inside the method.

```
Node prev = null;
```

Wrong, because it gets reinitialized on every recursive call.

Correct:

```
class Solution {
    Node prev = null;
}
```

---

### Mistake 5

Return `root` instead of `head`.

The caller needs the first node of the DLL, not the last processed node.

# Interview Questions

### 1. Why use inorder traversal?

Because the DLL must preserve the inorder sequence of the binary tree.

---

### 2. Why do we need `prev`?

To remember the previously visited node so we can link it with the current node.

---

### 3. Why do we need `head`?

The first node visited during inorder traversal becomes the head of the DLL, and we need to return it after the conversion.

---

### 4. Is a new DLL created?

No. The conversion is done **in-place** by reusing the existing `left` and `right` pointers.

---

### 5. Can this be done iteratively?

Yes. An iterative inorder traversal using a stack and a `prev` pointer also works with the same time complexity.
