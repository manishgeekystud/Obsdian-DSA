# Zigzag (Spiral) Level Order Traversal 
---

# Problem Statement

Given the **root of a Binary Tree**, return the **Zigzag (Spiral) Level Order Traversal**.

### Rule

- **Odd-numbered levels** → Traverse **Left → Right**
- **Even-numbered levels** → Traverse **Right → Left**

The traversal direction changes after every level.

**Examples:**

**Input:** root = [1, 2, 3, 4, 5, 6, 7] ![](https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/907336/Web/Other/blobid0_1754394121.webp)       
**Output:** [1, 3, 2, 4, 5, 6, 7]  
**Explanation:  
**Level 1 (left to right): [1]  
Level 2 (right to left): [3, 2]  
Level 3 (left to right): [4, 5, 6, 7]  
Final result: [1, 3, 2, 4, 5, 6, 7]

**Input:** root = [7, 9, 7, 8, 8, 6, N, 10, 9]  
 
**![](https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/907336/Web/Other/blobid2_1754394198.webp)**
**Output:** [7, 7, 9, 8, 8, 6, 9, 10]   
**Explanation:  
**Level 1 (left to right): [7]  
Level 2 (right to left): [7, 9]  
Level 3 (left to right): [8, 8, 6]  
Level 4 (right to left): [9, 10]  
Final result: [7, 7, 9, 8, 8, 6, 9, 10]

# Key Observation

This is simply **Level Order Traversal (BFS)** with one modification:

> Reverse the output of every alternate level.

The tree traversal itself **does not change**.

Only the **output order** changes.

---

# Why Can't We Change Queue Order?

Many beginners try:

```
Odd Level

Insert Left
Insert Right

Even Level

Insert Right
Insert Left
```

This is **wrong**.

Reason:

A Queue always follows **FIFO (First In First Out)**.

Changing the insertion order changes the order of the **next level**, not the current level.

So the zigzag pattern is not produced correctly.

---

# Correct Approach

1. Perform normal **Level Order Traversal**.
2. Store each level in a temporary list.
3. If the level is even, reverse the temporary list.
4. Add the temporary list to the final answer.

---

# Algorithm

```
Create Queue

Insert Root

leftToRight = true

While Queue is not empty

    Get current level size

    Create temporary list

    Process all nodes of current level

        Remove node

        Store node in temporary list

        Insert left child

        Insert right child

    If direction is Right → Left

        Reverse temporary list

    Add temporary list to answer

    Toggle direction
```

# Java Code (Queue + Reverse)

```
class Solution {

    ArrayList<Integer> zigZagTraversal(Node root) {

        ArrayList<Integer> result = new ArrayList<>();

        if (root == null)
            return result;

        Queue<Node> queue = new LinkedList<>();
        queue.offer(root);

        boolean leftToRight = true;

        while (!queue.isEmpty()) {

            int size = queue.size();

            ArrayList<Integer> temp = new ArrayList<>();

            for (int i = 0; i < size; i++) {

                Node curr = queue.poll();

                temp.add(curr.data);

                if (curr.left != null)
                    queue.offer(curr.left);

                if (curr.right != null)
                    queue.offer(curr.right);
            }

            if (!leftToRight)
                Collections.reverse(temp);

            result.addAll(temp);

            leftToRight = !leftToRight;
        }

        return result;
    }
}
```

# Complexity Analysis

### Time Complexity

Every node is visited exactly once.

Reversing alternate levels together still processes each node at most once.

```
O(N)
```

---

### Space Complexity

Queue stores at most one level.

Temporary list stores one level.

```
O(W)
```

where **W** is the maximum width of the tree.

In the worst case:

```
O(N)
```
