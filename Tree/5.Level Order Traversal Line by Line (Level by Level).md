This is an extension of **Level Order Traversal (BFS)** where each tree level is printed on a **new line**.

> **Interview Frequency:** ⭐⭐⭐⭐⭐ (Very Frequently Asked)

# Example

```
            10
          /    \
        20      30
       /  \    /  \
     40   50  60   70
          /
         80
```

### Output

```
10
20 30
40 50 60 70
80
```

---

# Idea

Normal Level Order Traversal prints everything in one line.

To print **level by level**, we need to know:

> **How many nodes belong to the current level?**

The queue already contains all nodes of the current level before we start processing that level.

So:

1. Store `queue.size()` in a variable.
2. Process exactly that many nodes.
3. Print a new line.
4. Repeat.

---

# Algorithm

```
1. Insert root into queue.

2. While queue is not empty

      size = queue.size()

      Repeat size times

            Remove node
            Print node

            Insert left child
            Insert right child

      Print new line
```

---

# Java Code (With Comments)

```java

public class LevelOrderLineByLine {

    static void levelOrder(Node root) {

        // Empty tree
        if (root == null)
            return;

        // Queue for BFS
        Queue<Node> queue = new LinkedList<>();

        // Insert root
        queue.offer(root);

        // Continue until queue becomes empty
        while (!queue.isEmpty()) {

            // Number of nodes in current level
            int size = queue.size();
            // Process current level
            for (int i = 0; i < size; i++) {

                // Remove front node
                Node current = queue.poll();
                // Print node
                System.out.print(current.data + " ");
                // Insert left child
                if (current.left != null)
                    queue.offer(current.left);
                // Insert right child
                if (current.right != null)
                    queue.offer(current.right);
            }

            // Level completed
            System.out.println();
        }
    }

  
}
```

# Why `size = queue.size()`?

Suppose the queue contains:

```
Queue

20 30
```

These two nodes are exactly the nodes of the **current level**.

While processing them, we insert their children:

```
40 50 60 70
```

If we don't save the initial `size`, the loop would continue processing these newly added nodes immediately, causing all levels to be printed on the same line.

By storing:

```
int size = queue.size();
```

we ensure we process **only the nodes that were already in the queue** for the current level.

---

# Time Complexity

Every node is visited exactly once.

```
Time Complexity = O(n)
```

---

# Space Complexity

The queue stores nodes of one level at a time.

- **Worst Case:** `O(n)` (complete binary tree)
- **Balanced Tree:** up to `O(n/2)` nodes in the last level, which is still `O(n)`.

---

# Interview Questions

### 1. Why use `queue.size()`?

To know exactly how many nodes belong to the current level before adding nodes from the next level.

### 2. Can this be done without `queue.size()`?

Yes. Another common approach uses a `null` marker (delimiter) after each level. When `null` is dequeued, print a newline and enqueue another `null` if more nodes remain. However, the `queue.size()` approach is cleaner and is generally preferred in interviews.

### 3. Is this still BFS?

Yes. It is the standard **Breadth-First Search (BFS)** traversal; the only addition is grouping nodes by their level.

while (!queue.isEmpty()) {

## Maximum Width of Binary Tree (Using Level Order Traversal)

### Definition

The **width of a binary tree** is the **number of nodes present at a particular level**.

The **maximum width** is the **maximum number of nodes among all levels**.

---

### Key Observation

Since width is calculated **level by level**, we use **Level Order Traversal (BFS)**.

At the start of every level,

```
int size = q.size();
```

`size` represents the **number of nodes in the current level**, which is the **width** of that level.

Update the maximum width:

```
maxWidth = Math.max(maxWidth, size);
```

Then process exactly `size` nodes before moving to the next level.

---

### Algorithm

1. If the tree is empty, return `0`.
2. Insert the root into the queue.
3. While the queue is not empty:
    - Store `size = q.size()` (current level width).
    - Update `maxWidth`.
    - Process all `size` nodes.
    - Add their left and right children to the queue.
4. Return `maxWidth`.

---

### Dry Run

```
        1
      /   \
     2     3
    / \   /
   4   5 6
```

|Level|Queue at Start|`size` (Width)|`maxWidth`|
|---|---|---|---|
|1|[1]|1|1|
|2|[2, 3]|2|2|
|3|[4, 5, 6]|3|3|

**Answer = 3**

---

### Time Complexity

```
O(n)
```

Each node is visited exactly once.

---

### Space Complexity

```
O(n)
```

In the worst case, the queue stores all nodes at the widest level.

---

### Common Mistake ❌

Do **not** calculate width after polling a node:

```
Node curr = q.poll();
width = q.size();   // ❌ Incorrect
```

After `poll()`, the queue size decreases by one, so it no longer represents the width of the current level.

✅ Always calculate width **before** processing the level:

```
int size = q.size();   // Correct width of current level
maxWidth = Math.max(maxWidth, size);
```
