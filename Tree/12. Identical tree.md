# Identical Tree (Same Tree)

> **Interview Frequency:** ⭐⭐⭐⭐⭐ (Very Frequently Asked)

This is one of the most common Binary Tree interview questions asked in **Amazon, Microsoft, Google, Walmart, Oracle, Adobe, Flipkart, TCS Digital**, etc.

---

# Problem Statement

Given the roots of **two binary trees**, determine whether they are **identical**.

Two binary trees are considered **identical** if:

1. They have the **same structure**.
2. The corresponding nodes contain the **same data**.

If both conditions are satisfied, return **true**; otherwise, return **false**.

# Idea

For two trees to be identical:

For every pair of corresponding nodes,

- Both should be `null`, **or**
- Both should contain the same value.

Then recursively compare

- Left subtree
- Right subtree

---

# Recursive Algorithm

```
isIdentical(root1, root2)

1. If both nodes are NULL
       return true

2. If one node is NULL
       return false

3. If data is different
       return false

4. Return

       isIdentical(left,left)

       AND

       isIdentical(right,right)
```

---

# Recursive Formula

```
isIdentical(r1,r2)

↓

Compare current nodes

↓

Compare Left Subtree

↓

Compare Right Subtree
```

---

# Complete Java Code (With Comments)

```java
class Node {

    int data;
    Node left;
    Node right;

    Node(int data) {
        this.data = data;
    }
}

public class IdenticalTree {

    static boolean isIdentical(Node r1, Node r2) {

        // Case 1: Both nodes are null
        if (r1 == null && r2 == null)
            return true;

        // Case 2: One node is null
        if (r1 == null || r2 == null)
            return false;

        // Case 3: Data is different
        if (r1.data != r2.data)
            return false;

        // Case 4: Compare left and right subtrees
        return isIdentical(r1.left, r2.left)
                && isIdentical(r1.right, r2.right);
    }

    public static void main(String[] args) {

        /*
                  Tree 1

                      1
                    /   \
                   2     3
        */

        Node root1 = new Node(1);
        root1.left = new Node(2);
        root1.right = new Node(3);

        /*
                  Tree 2

                      1
                    /   \
                   2     3
        */

        Node root2 = new Node(1);
        root2.left = new Node(2);
        root2.right = new Node(3);

        System.out.println(isIdentical(root1, root2));
    }
}
```

# Why Does This Work?

At every recursive call, we verify:

1. **Both nodes are null** → They match.
2. **One node is null** → Structures differ.
3. **Data values differ** → Trees differ.
4. **Current nodes match** → Recursively verify the left and right subtrees.

If any check fails, the function immediately returns `false`.

---

# Time Complexity

Each node is visited exactly once.

```
Time Complexity = O(n)
```

where `n` is the number of nodes (assuming both trees have the same number of nodes).

---

# Space Complexity

The extra space comes from the recursion stack.

- **Balanced Tree:** `O(log n)`
- **Skewed Tree:** `O(n)`

---

# Common Mistakes

### ❌ Mistake 1: Forgetting the "one node is null" check

```
if (r1.left == null)
```

This can throw a `NullPointerException` if `r1` itself is `null`.

Always check:

```
if (r1 == null || r2 == null)
    return false;
```

---

### ❌ Mistake 2: Comparing only child data

Incorrect:

```
if (r1.left.data != r2.left.data)
```

The recursive call will naturally compare child nodes when it reaches them.

---

### ❌ Mistake 3: Not comparing the root values

Always compare:

```
if (r1.data != r2.data)
    return false;
```

before recursing.

---

# Interview Questions

### 1. Why do we compare the current node before recursion?

If the current node values differ, there is no need to compare the subtrees because the trees are already different.

---

### 2. Why don't we compare `left.data` and `right.data` separately?

Because recursion handles that automatically. When the recursive call reaches the left child, it compares `r1.left.data` with `r2.left.data`, and similarly for the right child.

---

### 3. Can this be solved iteratively?

Yes. Use two queues (or two stacks), traverse both trees simultaneously, and compare corresponding nodes at each step.

---

### 4. Which traversal pattern does this use?

This solution follows a **preorder-style recursive pattern**:

```
Current Node
     ↓
Left Subtree
     ↓
Right Subtree
```

The current nodes are validated before exploring their children.

---

# Interview Tip

Many binary tree comparison problems follow this same recursive template:

```
if (both == null)
    return true;

if (one == null)
    return false;

if (current values differ)
    return false;

return leftResult && rightResult;
```

You'll use this pattern not only for **Identical Tree**, but also for problems like **Symmetric Tree**, **Subtree of Another Tree**, and **Flip Equivalent Binary Trees**.
