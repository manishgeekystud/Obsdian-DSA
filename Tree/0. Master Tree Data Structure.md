# Phase 4: Learn the Universal Recursive Template

Nearly every recursive tree solution follows this pattern.

```
int solve(Node root) {

    if(root == null)
        return base;

    int left = solve(root.left);

    int right = solve(root.right);

    // Process current node

    return something;
}
```

This one template solves many problems by changing:

- The base case
- The value returned
- The processing at the current node

# Common Mistakes

- Trying to memorize solutions instead of understanding patterns.
- Forgetting the base case (`root == null`).
- Mixing up preorder, inorder, and postorder.
- Not identifying whether the problem needs DFS or BFS.
- Ignoring time complexity and repeated work (for example, the `O(N²)` diameter solution).

# Interview Strategy

When you see a new tree problem, ask yourself these questions **in order**:

1. **Is it asking for traversal?**
    - DFS or BFS?
2. **Does it ask for levels, views, or width?**
    - Think **Queue (Level Order)**.
3. **Does the answer depend on children's results?**
    - Think **Postorder recursion**.
4. **Am I searching for a node or path?**
    - Think **DFS search**.
5. **Do I need to return a value or update a global answer?**
    - Return height/count if the parent needs it.
    - Use a global variable (or helper object) for the overall result.

---

## The Biggest Secret

After solving enough tree problems, you'll notice that **80–90% of interview questions are built from just a handful of recursive and BFS templates**. Focus on understanding _why_ each template works, practice dry runs, and you'll be able to adapt them to unfamiliar problems instead of memorizing individual solutions
