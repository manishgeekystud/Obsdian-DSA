# Maximum Path Sum in Binary Tree 

This is one of the **most frequently asked Binary Tree interview questions** in companies like **Amazon, Microsoft, Google, Adobe, Oracle, Walmart, Samsung, Flipkart, and Goldman Sachs**.

It is considered a **Tree Dynamic Programming (Tree DP)** problem because each node computes its answer using information from its children.

---

# Problem Statement

Given the root of a binary tree, find the **maximum path sum**.

A **path** is any sequence of nodes connected through parent-child edges.
**Examples:**

**Input:** root[] = [10, 2, 10, 20, 1, N, -25, N, N, N, N, 3, 4]
**Output:** 42
**Explanation:** Max path sum is represented using green colour nodes in the above binary tree.![](https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/700611/Web/Other/blobid3_1736948585.png)  

**Input:** root[] = [-17, 11, 4, 20, -2, 10]
**Output:** 31
**Explanation:** Max path sum is represented using green colour nodes in the above binary tree.![](https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/700611/Web/Other/blobid1_1736947534.png)  

**Constraints:**  
1 ≤ number of nodes ≤ 103  
-104 ≤ node->data ≤ 104
### Important Rules

- The path can start from **any node**.
- The path can end at **any node**.
- The path must be continuous.
- A node can appear only once.
- The path **does not have to pass through the root**.

# Observation

At every node there are **two different values**.

## 1. Maximum Gain (Return to Parent)

The parent can continue the path through **only one child**.

So return

```
Current Node

+

Maximum(left gain, right gain)
```

Formula

```
root.data + max(leftGain, rightGain)
```

---

## 2. Maximum Path Through Current Node

A complete path may use

- Left subtree
- Current node
- Right subtree

Formula

```
leftGain

+

root.data

+

rightGain
```

This path **cannot** be returned to the parent because it branches in two directions.

Instead, compare it with the global answer.

---

# Why Can't We Return Both Children?

Suppose

```
        10
       /  \
      2    8
```

Path through 10

```
2 → 10 → 8
```

If we return

```
2 + 10 + 8
```

to the parent,

the parent would create an invalid path with **three branches**.

A valid path cannot split.

Therefore, the parent must receive only **one branch**.

---

# Key Observation

Every recursive call performs **two tasks**.

### Task 1

Update global answer

```
maxSum = Math.max(maxSum,
                  root.data + leftGain + rightGain);
```

---

### Task 2

Return gain to parent

```
return root.data + Math.max(leftGain, rightGain);
```

This pattern is exactly the same as **Diameter of Binary Tree**.

---

# Algorithm

```
Compute Left Gain

↓

Compute Right Gain

↓

Ignore Negative Gains

↓

Update Global Maximum

↓

Return Best Gain to Parent
```

---

# Why Ignore Negative Paths?

Suppose

```
        5
       /
    -20
```

Including

```
-20
```

reduces the path sum.

Instead,

ignore it.

Code

```
int left = Math.max(0, helper(root.left));
```

If left subtree contributes a negative value,

use

```
0
```

instead.

---

# Java Code

```
class Solution {

    int maxSum = Integer.MIN_VALUE;

    int findMaxSum(Node root) {

        helper(root);

        return maxSum;
    }

    int helper(Node root) {

        if (root == null)
            return 0;

        int leftGain = Math.max(0, helper(root.left));
        int rightGain = Math.max(0, helper(root.right));

        maxSum = Math.max(maxSum,
                          root.data + leftGain + rightGain);

        return root.data + Math.max(leftGain, rightGain);
    }
}
```

# Common Mistakes

### Mistake 1

Returning

```
root.data + left + right
```

Wrong.

A parent cannot continue a path through both children.

Correct

```
root.data + Math.max(left, right)
```

---

### Mistake 2

Not ignoring negative gains.

Wrong

```
int left = helper(root.left);
```

Correct

```
int left = Math.max(0, helper(root.left));
```

---

### Mistake 3

Returning `maxSum`.

The helper function should return the **maximum gain**, not the final answer.

The final answer comes from

```
findMaxSum()
```

---

### Mistake 4

Initializing

```
int maxSum = 0;
```

Wrong.

If every node is negative,

the answer becomes incorrect.

Correct

```
int maxSum = Integer.MIN_VALUE;
```

# Interview Questions

### 1. Why can't we return `left + root + right`?

Because a path cannot split into two branches when extending to a parent. The parent can continue through only one child.

---

### 2. Why do we ignore negative gains?

A negative path only decreases the total sum, so it's better to exclude it by treating its contribution as `0`.

---

### 3. Why use a global variable?

The recursive function returns the best gain to the parent, but the maximum path can end at any node. A global variable tracks the best path seen anywhere in the tree.

---

### 4. Why initialize `maxSum` with `Integer.MIN_VALUE`?

If all node values are negative, initializing to `0` would incorrectly return `0`. Using `Integer.MIN_VALUE` ensures the algorithm correctly handles all-negative trees.
