The **Children Sum Property** means:

> For every non-leaf node, the value of the node should be equal to the **sum of its left child and right child**.

```
Node.data = LeftChild.data + RightChild.data
```

If a child is missing, its value is considered **0**.

> **Interview Frequency:** ⭐⭐⭐⭐⭐ (Very Frequently Asked)

---

# Example 1 (Valid Tree)

```
            20
          /    \
         8      12
        / \    /  \
       3   5  7    5
```

Let's verify:

```
20 = 8 + 12   ✔

8 = 3 + 5     ✔

12 = 7 + 5    ✔
```

Leaf nodes do not need checking.

**Output**

```
Children Sum Property = True
```

# Recursive Idea

For every node:

1. Check left subtree.
2. Check right subtree.
3. Check

```
Node.data == Left.data + Right.data
```

If **all** are true, return `true`.

---

# Algorithm

```
If node is NULL
    return true

If node is leaf
    return true

left = value of left child (0 if absent)

right = value of right child (0 if absent)

return

(node.data == left + right)
AND
childrenSum(left subtree)
AND
childrenSum(right subtree)
```

---

# Java Code (With Comments)

```
class Node {

    int data;
    Node left, right;

    Node(int data) {
        this.data = data;
    }
}

public class ChildrenSumProperty {

    static boolean isChildrenSum(Node root) {

        // Empty tree satisfies property
        if (root == null)
            return true;

        // Leaf node also satisfies property
        if (root.left == null && root.right == null)
            return true;

        // Left child value
        int left = 0;

        if (root.left != null)
            left = root.left.data;

        // Right child value
        int right = 0;

        if (root.right != null)
            right = root.right.data;

        // Check current node and recursively check subtrees
        return (root.data == left + right)
                && isChildrenSum(root.left)
                && isChildrenSum(root.right);
    }

    public static void main(String[] args) {

        /*
                    20
                  /    \
                 8      12
                / \    /  \
               3   5  7    5
        */

        Node root = new Node(20);

        root.left = new Node(8);
        root.right = new Node(12);

        root.left.left = new Node(3);
        root.left.right = new Node(5);

        root.right.left = new Node(7);
        root.right.right = new Node(5);

        System.out.println(isChildrenSum(root));
    }
}
```

true

# Time Complexity

Each node is visited once.

```
Time Complexity = O(n)
```

where `n` is the number of nodes.

---

# Space Complexity

Depends on the recursion stack.

- **Balanced Tree:** `O(log n)`
- **Skewed Tree:** `O(n)`

# Common Interview Questions

### 1. Why do leaf nodes return `true`?

Leaf nodes have no children, so there is no sum to verify. By definition, they satisfy the property.

---

### 2. Why does a `null` node return `true`?

An empty tree cannot violate the property, making `true` the correct base case.

---

### 3. Why use `0` for a missing child?

If only one child exists, the missing child's contribution is `0`.

Example:

```
     15
    /
   15
```

```
Left = 15

Right = 0

15 = 15 + 0 ✔
```

# Interview Tip

Remember the recursive pattern for property-checking problems:

```
return currentNodeCondition
       && leftSubtreeCondition
       && rightSubtreeCondition;
```

Examples:

- **Children Sum Property**
- Identical Trees
- Symmetric Tree
- Balanced Tree (optimized version)
- Validate Binary Search Tree

This pattern is one of the most common recursion patterns used in binary tree interview questions.
